Detailed Description

Allows multiple threads to wait until the condition is set, acting like a gate.

As long as the condition remains set, any threads that wait on the condition will immediately proceed. The condition must manually be reset in order to cause threads/processes to wait until the next time the condition is set.

Sets the condition to true and wakes up threads.

Sets the condition's internal state to true and wakes up any threads waiting on the condition. The condition will remain in the set state until it is manually reset.

◆ reset()

void cpen333::thread::condition::reset ( )

Resets the condition.

Sets the condition's internal state to false. This will call any future wait calls to block until the condition is again notified.

void cpen333::thread::condition::wait ( )

Waits until the condition is set.

Causes the current thread to block until the condition is set. This condition will not exhibit spurious wake-ups. A thread will be forced to wait here indefinitely until the condition is set.
